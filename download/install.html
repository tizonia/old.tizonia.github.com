Clone the repos

Tizonia OpenMAX IL

Clone the Tizonia OpenMAX IL repo. You can choose to do it from GitHub or Bitbucket. 

      # On Bitbucket, clone the Mercurial repo...
      $ hg clone https://bitbucket.org/tizonia/tizonia-openmax-il

      # On GitHub, clone the Git repo...
      $ git clone https://github.com/tizonia/tizonia-openmax-il.git

Skema, Tizonia's test execution framework

Clone the skema test execution framework. Again, you can choose to do it from GitHub or Bitbucket. 

      # On Bitbucket, clone the Mercurial repo...
      $ hg clone https://bitbucket.org/tizonia/skema

      # On GitHub, clone the Git repo...
      $ git clone https://github.com/tizonia/skema.git

Download the test media files

The unit tests use a number of Creative Commons licensed media files that you can download using the following command: 

      $ wget http://www.juanrubio.me/wp-content/uploads/2012/05/tizonia_test_media.tgz

Install the dependencies

Use the following command to install the dependencies (this assumes a Debian-based Linux system): 

      # Open a shell and execute the following...
      $ sudo apt-get install autoconf automake autotools-dev build-essential check cscope doxygen indent wget libtool liblog4c-dev liblog4c-doc liblog4c3 libasound2 libasound2-dev libdbus-1-dev libdbus-c++-1-0 libdbus-c++-dev libsqlite3-dev libboost-all-dev python-sphinx sqlite3 uuid uuid-dev libsdl1.2debian libsdl1.2-dev libmad0 libmad0-dev  libvpx-dev

Build and install the software

Install the Tizonia libraries somewhere in your system E.g., type the following to install the libraries and data under $HOME/temp/tizonia: 

      # e.g., for a a 'debug-type' of build, type the following commands...
      $ mkdir ~/temp
      $ autoreconf -ifs ; ./configure --enable-silent-rules --prefix=/home/$USER/temp/tizonia CFLAGS="-ggdb -Wall -Werror"
      $ make -s j4 V=0
      $ make install

Get the config files ready

A number of configuration files need to be in place before Tizonia OpenMAX IL is ready to be used.

Tizonia's RM D-Bus service file (mandatory)

Copy Tizonia RM D-Bus service file to one of the D-Bus search folders in your system. 

      # e.g.: Copy the service file to $HOME/.local/share/dbus-1/services
      $ mkdir -p ~/.local/share/dbus-1/services
      $ cp ~/temp/share/dbus-1/services/com.aratelia.tiz.rm.service ~/.local/share/dbus-1/services

Tizonia rc file (mandatory)

Copy Tizonia's rc file to your $HOME, e.g. 

      $ cp 'path_to_tizonia_repo'/libtizosal/src/tizrc ~/.tizrc

Edit the $HOME/.tizrc file and replace all instances of user with your user name: 

      $ sed -i "s/user/$USER/g" ~/.tizrc

At this stage, it is a good idea to verify that all the paths described in the Tizonia rc file point to the correct locations in your system (this should not be necessary if you followed all the above steps without modifications).

Copy or update the log4c rc file (optional)

Tizonia uses the log4c logging library to output debug information. If you want to use this facility, just create/update the log4crc file in your $HOME directory. E.g: 

      $ cp 'path_to_tizonia_repo'/libtizosal/src/log4crc ~/.log4crc

And customize this file to your needs.

Run the unit tests

And finally, run the unit tests to verify that things work. 

      # 'cd' into the Tizonia repo and type:
      $ make check
